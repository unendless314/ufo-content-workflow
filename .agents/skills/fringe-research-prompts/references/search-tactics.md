# 領域特定搜索策略

根據主題類型選擇對應的搜索策略。

---

## 按阻力類型分類

### Type A: 安全/機密主題
**主題**：UFO/UAP、生物武器、黑預算項目、冷戰秘密項目
**阻力**：Classification（分級保密）

**搜索策略：**
```
site:theblackvault.com
site:nsarchive.gov
filetype:pdf "classification guide"
filetype:pdf "redacted"
"SAP" "Special Access Program"
FOIA "Freedom of Information Act"
"declassified" "formerly classified"
"congressional hearing" "closed session"
```

**關鍵來源：**
- The Black Vault（解密文件庫）
- National Security Archive
- Congressional Record（聽證紀錄）
- 官方解密報告（如 UAPTF、AARO 報告）

---

### Type B: 科學/典範主題
**主題**：超能力 (Psi)、冷核融合、意識研究、替代能源
**阻力**：Dogma/Theory（與現有典範衝突）

**搜索策略：**
```
site:arxiv.org "anomalous"
"Journal of Scientific Exploration"
"anomalous effect size"
"replication failure"
"file drawer problem"
"publication bias"
site:parapsych.org
"Ganzfeld experiment"
"remote viewing" "Stargate Project"
```

**關鍵策略：**
- 尋找 "Failed Debunkings"（懷疑論者試圖複製卻發現異常，但將其否認）
- 檢查被撤稿論文的後續爭議
- 尋找「負面結果」的元分析

**關鍵來源：**
- Journal of Scientific Exploration
- ArXiv（邊緣物理學預印本）
- Parapsychological Association 文獻
- 冷戰時期解密的心靈研究（如 Stargate Project）

---

### Type C: 歷史/時間線主題
**主題**：古代失落技術、異常年代測定、失落文明
**阻力**：Timeline Inertia（挑戰既定歷史敘事）

**搜索策略：**
```
"out of place artifact" OOPArt
"anomalous dating"
"omitted artifacts"
"forbidden archaeology"
site:jstor.org "anomalous"
"pre-Clovis" "archaeological controversy"
"Göbekli Tepe" "rewrite history"
"Baghdad Battery" "Antikythera mechanism"
```

**關鍵策略：**
- 尋找被主流學術忽略的發現
- 檢查新發現如何「被迫適應」現有時間線
- 關注地質學與考古學的日期衝突

**關鍵來源：**
- 學術期刊的「爭議」欄目
- 考古學會議紀錄中的異議
- 獨立研究者的詳細技術分析

---

### Type D: 文化/污名主題
**主題**：神秘動物學 (Cryptozoology)、通靈現象
**阻力**：Ridicule/Stigma（被視為迷信或笑話）

**搜索策略：**
```
"cryptid" "eyewitness testimony"
"giggle factor" UFO
"stigma" "scientific taboo"
"anomalous phenomenon" "peer review"
"scientific controversy"
```

**關鍵策略：**
- 尋找科學家私下的興趣 vs 公開的否認
- 關注證人的「社會代價」敘述
- 檢查媒體如何將主題「笑話化」

---

## Deep Dive Trigger 操作指南

**何時觸發：**
當搜索結果被「Debunking」「Myth」「Hoax」類文章主導時。

**強制轉向搜索：**
```
"Criticism of [Debunker Name]"
"Rebuttal to [Standard Explanation]"
"Problems with [Mainstream Theory]"
"[Expert Name] responds"
"[Case Name] inconsistencies"
```

**目標：**
找到被壓制的異議聲音，而非接受主流解釋的表面共識。
